Front-office

La solution s'appuie sur Hugo (https://gohugo.io), qui est un générateur open source de sites statiques. Cela présente plusieurs avantages :

  • sobriété numérique (https://diagnostic.noesya.coop/76d26fc7-0405-45c8-9a5f-19d3b5f83483)
  • accessibilité (bonne conformité RGAA)
  • sécurité (pas de langage serveur, pas de BDD, donc pas de risque de hack par la couche applicative)
  • souveraineté (paquet HTML CSS JS hébergeable de façon très souple)
  • ergonomie (édition simple, vocabulaire proche des logiques universitaires)
  • facilité de migration depuis WordPress (import automatique)
  • respect des données privées (conformité RGPD)
  • souveraineté (Scalingo et Scaleway sont des structures de droit français utilisant des datacenters en france)

Back-office

La solution s'appuie sur le langage open source Ruby, et le framework open source Ruby on Rails. Nous proposons une instance hébergée chez Scalingo et Scaleway, qui présente les avantages suivants :

  • ergonomie (édition simple, vocabulaire proche des logiques universitaires)
  • facilité de migration depuis WordPress (import automatique)- respect des données privées (conformité RGPD) 
  • souveraineté (Scalingo et Scaleway sont des structures de droit français utilisant des datacenters en france)

Limiter le Shadow IT

Que ce soit pour des préférences ergonomiques personnelles ou pour accélérer la mise en œuvre, les décisions prises par les métiers hors du périmètre de la DSI génèrent des risques de sécurité, des données doublonnées, et sont une cause importante de coûts cachés.
